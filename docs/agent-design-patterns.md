# Agent Design Patterns

How to write effective SOUL.md files for OpenClaw agents.

## The Standard Structure

Based on the [agency-agents](https://github.com/msitarzewski/agency-agents) standard:

### 1. Identity and Tagline
Start with a name and a punchy one-liner that captures the essence.

### 2. Who I Am
2-3 sentences. Role, personality, what makes this agent unique. Not "I am a helpful assistant" — be specific.

### 3. Core Principles
3-6 bold principles that define how the agent operates. These are the non-negotiable rules.

### 4. How I Work
Step-by-step process the agent follows. Numbered list. Clear sequence.

### 5. Domain-Specific Sections
The meat — methodology, standards, frameworks specific to the role.

### 6. Communication Style
1-2 sentences on tone. Examples help.

### 7. Shared Context
Which files to read and write. How this agent connects to the team.

### 8. Team Integration
Which topic, which teammates, how bot-to-bot communication works.

### 9. Learning and Memory
What the agent should remember and improve on over time.

### 10. Success Metrics
Measurable outcomes. Specific numbers. Not "do a good job" but "95% accuracy on triage."

## Key Principles

**Strong personality > generic helpfulness.** "I default to skepticism" beats "I help test things."

**Specific deliverables > vague promises.** Include formats, templates, and examples.

**Measurable metrics > subjective quality.** "Under 1.5s load time" beats "fast pages."

**Proven workflows > theoretical approaches.** Step-by-step, battle-tested.

## Anti-Patterns

- Generic "I am a helpful AI" personality
- Vague scope without boundaries
- No success metrics
- Missing escalation rules
- No team integration section
